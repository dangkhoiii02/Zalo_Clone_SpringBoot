# Zalo Clone Project

A comprehensive real-time messaging and video calling application built with Clean Architecture, inspired by WhatsApp and Zalo.

![Architecture Diagram](https://img.shields.io/badge/Architecture-Clean-brightgreen?style=flat-square) ![Backend Status](https://img.shields.io/badge/Backend-Go_1.26-blue?style=flat-square)

## System Architecture

The overarching system is divided into two primary parts:
1. **Backend Server** (Written in Golang using Clean Architecture)
2. **UI Application** (Future Flutter Client)

The infrastructure consists of several interconnected pieces deployed via `docker-compose`:
- **PostgreSQL**: For persistent tabular data (Users profiles, relationships, and authentication)
- **MongoDB**: For unstructured/high-volume data (Chats, Messages, Media metadata)
- **Redis**: For caching and real-time presence (Online status tracking using TTLs)
- **LiveKit Server (WebRTC SFU)**: Powers real-time low latency video and audio calling.

## Features

- **Authentication & Security:** JWT-based authentications, rate limiting, payload validations.
- **Real-Time Data Flow:** Powered natively by WebSockets.
- **RESTful Endpoints:** Swagger UI integration for interactive documentation.
- **Message System Integration:** Typing indicators, Read receipts, History pagination.
- **Peer-to-Peer Calling:** Handled using LiveKit's token routing mechanism (Video/Audio).

## Getting Started

### 1. Prerequisites

Please ensure that you have installed the following requirements on your local machine:
- `Go 1.25+`
- `Docker` & `Docker Desktop / docker-compose`

### 2. Setting Up the Environment

1. Rename or create an `.env` file in the root directory and update it according to your needs:
   ```env
   # PostgreSQL
   POSTGRES_USER=zalouser
   POSTGRES_PASSWORD=zalosecret
   POSTGRES_DB=zalodb
   POSTGRES_HOST=localhost
   POSTGRES_PORT=5432
   
   # MongoDB
   MONGO_URI=mongodb://localhost:27017
   MONGO_DB=zalochat
   
   # Redis
   REDIS_ADDR=localhost:6379
   
   # JWT & Security
   JWT_SECRET=zalo-clone-super-secret-key-change-in-production-2026
   JWT_EXPIRY=24h
   ```

2. Start up the backend infrastructure:
   ```bash
   docker-compose up -d
   ```

### 3. Running the Server

Navigate to the Go backend folder and download the packages:
```bash
cd backend
go mod tidy
go run ./cmd/server/
```

The Go app will perform database migrations on start up, connect to external services, and expose the main router HTTP server on `0.0.0.0:8080`.

### 4. API Documentation (Swagger)

Once the server is running, the Swagger interactive UI is served at:
👉 **[http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html)**

You can use the built-in Swagger client to interface with User Authentication, Profile Updates, WebRTC tokens generation, and the Friendship API.

## Project Structure (Go)

```text
backend/
├── cmd/server/            # Entry point for the executable
├── docs/                  # Autogenerated Swagger specs
├── internal/
│   ├── config/            # Core .env and global configs
│   ├── delivery/          # Transport layer implementations (HTTP/Gin, WebSockets)
│   ├── domain/            # Plain-text Structs mappings, business logic variables
│   ├── infrastructure/    # Concrete DB/external configs initialization
│   ├── middleware/        # Global router middlewares
│   ├── repository/        # PostgreSQL, Redis, Mongo concrete implementations
│   └── usecase/           # Primary application logic and service layers
└── go.mod                 # Go modules metadata
```
